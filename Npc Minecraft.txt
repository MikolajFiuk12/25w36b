## **1. Co to są komendy w Minecraft?**

Komendy w Minecraft to **specjalne polecenia wpisywane w grze**, które pozwalają Ci zmieniać świat, zarządzać graczami, mobami, blokami i innymi elementami gry w sposób natychmiastowy.
Można je wpisywać w:

* czacie (wpisujesz `/komenda`)
* command blockach (blokach specjalnie do automatyzacji komend)

Komendy pozwalają na **praktycznie wszystko**, od teleportacji, przez tworzenie struktur, po generowanie efektów specjalnych i niestandardowych mobów.

---

## **2. Po co używa się komend?**

Komendy są użyteczne w wielu sytuacjach:

1. **Dla gracza** – szybko zdobywasz przedmioty, zmieniasz pogodę, teleportujesz się lub ustawiasz punkt odrodzenia.
2. **Dla map twórców i serwerów** – tworzenie przygodowych map, minigier, wyzwań i automatycznych systemów.
3. **Dla command blocków** – można tworzyć automatyczne zdarzenia w grze, np.:

   * teleportacja mobów w określone miejsce
   * spawn lootu w chestach
   * pokazywanie napisów i efektów wizualnych
   * systemy punktów i nagród w minigrach
4. **Dla eksperymentów** – testowanie funkcji, modyfikowanie świata bez granic, tworzenie machin redstone z automatyką

---

## **3. Przykłady, co możesz robić dzięki komendom**

### **Teleportacja i ruch**

* Teleportacja gracza lub mobów w dowolne miejsce:

```
/tp @p 100 64 -200
```

* Tworzenie „portali” lub automatycznych przesunięć graczy w mapach przygodowych.

---

### **Generowanie przedmiotów i mobów**

* Daj graczowi dowolny przedmiot z enchantami i właściwościami:

```
/give @p minecraft:diamond_sword{Enchantments:[{id:"minecraft:sharpness",lvl:5}]}
```

* Przywołanie custom mobów:

```
/summon minecraft:zombie ~ ~ ~ {CustomName:"\"Strażnik\""}
```

* Możesz tworzyć bossy, niestandardowe moby, a nawet niewidzialne NPC do map przygodowych.

---

### **Zmiana świata**

* Zmiana czasu i pogody:

```
/time set day
/weather thunder
```

* Wypełnianie bloków i tworzenie struktur:

```
/fill ~ ~ ~ ~10 ~5 ~10 minecraft:stone
```

* Możesz szybko budować ogromne struktury lub niszczyć teren.

---

### **Efekty i wizualizacje**

* Efekty wizualne, cząsteczki i dźwięki:

```
/particle minecraft:flame ~ ~1 ~
/playsound minecraft:entity.ender_dragon.growl @a
```

* Możesz robić widowiskowe efekty w minigrach lub mapach przygodowych.

---

### **Scoreboard i punkty**

* Tworzenie systemów punktów, nagród i rankingów:

```
/scoreboard objectives add Points dummy "Punkty"
```

* Możesz zrobić gry typu wyścigi, labirynty z punktami lub mini-questy.

---

### **Command block – automatyzacja**

* Możesz łączyć komendy w **command blockach**, żeby tworzyć automatyczne systemy:

  * teleportacja co tick mobów do arena
  * automatyczne spawn lootu w chestach co X sekund
  * wyświetlanie napisów, tytułów i efektów wizualnych w momencie zdarzenia
  * tworzenie pułapek i wyzwań w mapach przygodowych

---

### **4. Dlaczego komendy są tak potężne?**

* Pozwalają **łamać ograniczenia normalnej gry** – np. dać nieskończone przedmioty, teleportować się w powietrzu, czy modyfikować moby w nierealny sposób.
* W command blockach można **łączyć komendy w logiczne systemy** – można praktycznie stworzyć własną mini grę w Minecraft bez żadnego moda.
* Dzięki nim można też **uczyć się logiki programowania** – komendy typu `execute if`, `scoreboard` czy `tag` działają jak warunki, zmienne i funkcje w programowaniu.

## **Przykładowa szczegółowa lista komend z opisem**

### 1. `/give`

* **Opis:** Daje graczowi przedmiot w określonej ilości i z opcjonalnymi właściwościami NBT.
* **Składnia:**

```
/give <gracz> <przedmiot> [ilość] [data] [NBT]
```

* **Argumenty:**

  * `<gracz>` – nazwa gracza lub selector (`@p`, `@a`, `@r`, `@e`)
  * `<przedmiot>` – np. `minecraft:diamond_sword`
  * `[ilość]` – liczba przedmiotów (domyślnie 1)
  * `[data]` – wariant przedmiotu (rzadko używane)
  * `[NBT]` – właściwości przedmiotu, np. enchanty, nazwa, lore
* **Przykład:**

```
/give @p minecraft:diamond_sword{Enchantments:[{id:"minecraft:sharpness",lvl:5}]} 1
```

Daje graczowi miecz diamentowy z Sharpness V.

---

### 2. `/tp`

* **Opis:** Teleportuje gracza lub mob do innego gracza lub współrzędnych.
* **Składnia:**

```
/tp <cel>  
/tp <gracz> <cel>  
/tp <gracz> <x> <y> <z> [yaw] [pitch]
```

* **Argumenty:**

  * `<cel>` – gracz lub mob, do którego ma nastąpić teleportacja
  * `<x y z>` – współrzędne docelowe
  * `[yaw pitch]` – kierunek, w którym będzie patrzył gracz
* **Przykład:**

```
/tp @p 100 64 -200 90 0
```

Teleportuje gracza do współrzędnych X=100, Y=64, Z=-200 i ustawia go w kierunku 90° yaw i 0° pitch.

---

### 3. `/summon`

* **Opis:** Przywołuje moby lub custom entity w wybranym miejscu z opcjonalnymi właściwościami NBT.
* **Składnia:**

```
/summon <mob> [x y z] [dataTag]
```

* **Argumenty:**

  * `<mob>` – np. `minecraft:zombie`, `minecraft:armor_stand`
  * `[x y z]` – współrzędne spawnienia
  * `[dataTag]` – właściwości mobów, np. wygląd, enchanty, AI
* **Przykład:**

```
/summon minecraft:armor_stand ~ ~ ~ {CustomName:"\"MójStojak\"",NoGravity:1b,Invisible:1b}
```

Tworzy niewidzialny armor stand z nazwą „MójStojak”, który nie spada.

---

### 4. `/execute`

* **Opis:** Najpotężniejsza komenda do wykonywania złożonych akcji i warunków.
* **Składnia:**

```
/execute [as <target>] [at <target>] [positioned <x y z>] [rotated <yaw pitch>] [if|unless <warunek>] run <komenda>
```

* **Przykład:**

```
/execute as @e[type=zombie] at @s if block ~ ~-1 ~ minecraft:grass_block run tp @s ~ ~10 ~
```

Każdy zombie stojący na trawie zostanie teleportowany 10 bloków w górę.

## **Command block basics – podstawy**

* `/give <gracz> command_block` – daje command block
* `/setblock <x y z> minecraft:command_block` – ustawia command block w świecie
* Typy command blocków:

  * **Impulse (pomarańczowy)** – wykonuje komendę jeden raz po aktywacji
  * **Repeat (fioletowy)** – wykonuje komendę co tick, dopóki jest aktywny
  * **Chain (zielony)** – wykonuje komendę tylko po wykonaniu poprzedniego command blocka
* **Conditional** – command block wykonuje komendę tylko jeśli poprzedni zakończył się sukcesem
* **Unconditional** – command block wykonuje komendę zawsze
* **Redstone** – command block wymaga sygnału redstone
* **Always Active** – command block działa bez sygnału redstone

## **Command block / execute commands**

Command blocky najczęściej używają **execute**, żeby robić zaawansowane akcje:

* `/execute as <target>` – wykonuje komendę jako inny gracz/mob
* `/execute at <target>` – wykonuje komendę w miejscu targetu
* `/execute positioned <x y z>` – wykonuje komendę w określonych współrzędnych
* `/execute rotated <yaw> <pitch>` – wykonuje komendę z określoną rotacją
* `/execute facing <x y z|entity>` – wykonuje komendę w kierunku
* `/execute if <warunek> run <komenda>` – wykonuje komendę jeśli warunek prawdziwy
* `/execute unless <warunek> run <komenda>` – wykonuje jeśli warunek fałszywy
* `/execute store result <target> <path> <typ>` – zapisuje wynik komendy do zmiennej NBT
* `/execute store success <target> <path> <typ>` – zapisuje status powodzenia komendy

Mannequins
25w36b
Added a new type of technical entity called minecraft:mannequin which can only be spawned with summon commands.

A Mannequin is a Player Avatar without a connected Player
Mannequins always show an "NPC" text where a Player's below_score score would show
Mannequins function as Living Entities - they can hold and wear equipment, have attributes & effects, take damage, etc
Data Fields:

profile - Which profile to show on the entity. This is either a Player profile (same format as a minecraft:profile component) or a Mannequin profile with the following fields:
texture - Namespaced ID of the skin texture to use
The skin is specified relative to the textures folder and with a .png suffix
e.g. entity/player/wide/steve will use the default wide Steve skin
cape - Optional namespaced ID of the cape texture to use
When specified, uses the same format as the texture field
Omitting this field means the Mannequin has no cape
elytra - Optional namespaced ID of the elytra texture to use
When specified, uses the same format as the texture field
Omitting this field means the Mannequin uses the cape texture, or if that is also omitted, the default Elytra texture when wearing Elytra
model - Optional model type, one of wide and slim
If omitted, wide is used
hidden_layers - List of outer skin layers to hide
Valid entries: cape, jacket, left_sleeve, right_sleeve, left_pants_leg, right_pants_leg, hat
main_hand - Which hand is the main hand of the Mannequn - one of left and right
