Okej — poniżej masz **pełną, krok‑po‑kroku instrukcję** jak przygotować pendrive instalacyjny z **Debianem 12** (wersja instalatora). Zamieściłem metody dla **Windows**, **Linux** i **macOS**, sprawdzenie sumy kontrolnej i kilka tipów do BIOS/UEFI. Zrób to uważnie — wszystkie dane na pendrive zostaną usunięte.

# 1) Pobranie obrazu ISO Debiana 12

1. Wejdź na oficjalną stronę Debiana i pobierz odpowiedni obraz, np. **netinst** (mały instalator) lub pełny obraz live/amd64:

   * wybierz architekturę `amd64` jeśli masz zwykły PC/laptop.
2. Pobierz też plik z sumami kontrolnymi (SHA256) z tej samej strony — będziesz weryfikować plik.

# 2) Weryfikacja sumy kontrolnej (zalecane)

Po pobraniu sprawdź SHA256, żeby upewnić się, że plik nie jest uszkodzony:

* Na Linux/macOS:

```bash
sha256sum debian-12.*.iso
# porównaj wynik z wartością w pobranym pliku SHA256SUMS
```

* Na Windows możesz użyć PowerShell:

```powershell
Get-FileHash .\debian-12.*.iso -Algorithm SHA256
```

Porównaj z oryginalną sumą ze strony Debiana.

# 3) Metoda A — Windows (najprościej: Rufus)

1. Pobierz **Rufus** (portable) na Windows.
2. Włóż pendrive (min. 4 GB dla netinst, 8 GB dla większych obrazów).
3. Uruchom Rufus:

   * Device: wybierz swój pendrive.
   * Boot selection: wybierz pobrany plik `.iso`.
   * Partition scheme: jeśli komputer używa UEFI — `GPT`, dla starszych BIOSów `MBR`.
   * Target system: Rufus dobierze automatycznie.
   * Kliknij **Start** → potwierdź usunięcie danych → poczekaj.
4. Po skończeniu bezpiecznie odłącz pendrive.

Rufus automatycznie ustawi obraz w trybie UEFI/BIOS i (zwykle) obsłuży Secure Boot.

# 4) Metoda B — Linux (dd — ostrożnie)

1. Włóż pendrive i znajdź jego urządzenie:

```bash
lsblk
# lub
sudo fdisk -l
```

Zlokalizuj np. `/dev/sdb` (nie `/dev/sdb1` — wskazujemy całe urządzenie). **Uwaga: pomyłka usunie dane na niewłaściwym dysku.**

2. Odmontuj partycje pendrive (jeśli są zamontowane):

```bash
sudo umount /dev/sdb1  # powtórz dla innych partycji
```

3. Nagraj ISO (przykład):

```bash
sudo dd if=/ścieżka/do/debian-12.xx-amd64-netinst.iso of=/dev/sdb bs=4M status=progress oflag=sync
```

* `bs=4M` i `oflag=sync` polepszają zapis i synchronizację.
* Poczekaj aż `dd` zakończy — nie przerywaj.

4. Po zakończeniu:

```bash
sync
sudo eject /dev/sdb
```

Wyjmij pendrive.

Alternatywy (łatwiejsze GUI): **balenaEtcher**, **Ventoy** (jeśli chcesz mieć wiele ISO na jednym pendrive).

# 5) Metoda C — macOS (dd)

Procedura jak w Linux:

1. Zlokalizuj pendrive:

```bash
diskutil list
```

2. Odmontuj:

```bash
diskutil unmountDisk /dev/diskN
```

3. Nagraj ISO (użyj `/dev/rdiskN` dla szybszego zapisu):

```bash
sudo dd if=/ścieżka/debian-12.iso of=/dev/rdiskN bs=4m status=progress
sync
```

4. Po zakończeniu:

```bash
diskutil eject /dev/diskN
```

# 6) Ventoy — pendrive z wieloma ISO (polecam)

Jeśli chcesz trzymać kilka instalatorów (np. różne wersje, narzędzia ratunkowe), użyj **Ventoy**:

1. Zainstaluj Ventoy na pendrive (program formatuje pendrive).
2. Po instalacji po prostu kopiuj pliki `.iso` na pendrive — przy starcie wybierzesz które ISO uruchomić.

# 7) BIOS / UEFI — uruchomienie instalatora

1. Włóż pendrive do docelowego komputera.
2. Wejdź do BIOS/UEFI (klawisze: F2, Del, F12, Esc — zależy od producenta).
3. Ustaw bootowanie z USB (UEFI lub Legacy/CSM w zależności od obrazu).
4. (Opcjonalnie) Jeśli problem z uruchomieniem, wyłącz tymczasowo **Secure Boot** lub użyj ustawienia kompatybilności Legacy, jeżeli instalator nie startuje. Debian 12 zwykle działa z UEFI i Secure Boot, ale niektóre obrazy/sterowniki mogą sprawiać problemy.

# 8) Najczęstsze problemy i rozwiązania

* **Pendrive nie bootuje** → sprawdź czy ISO poprawnie nagrane; spróbuj Rufus (Windows) lub Etcher (Linux/macOS).
* **Wybrałeś złe urządzenie dla `dd`** → niestety usuwa dane — uważaj na `of=/dev/sdX`.
* **Instalator nie widzi dysku NVMe** → może brak sterownika; spróbuj innego obrazu (live) lub sprawdź w BIOS ustawienia SATA/NVMe.
* **Chcesz persistent USB (live z zapisem)** → użyj narzędzi typu Rufus (persistent) lub tworzenia partycji persistent (zaawansowane).

# 9) Szybkie przypomnienie bezpieczeństwa

* Kopia ważnych danych z pendrive przed formatem.
* Upewnij się, że wskazujesz właściwe urządzenie przy `dd`.
* Weryfikuj SHA256 pobranego ISO.

# **2️⃣ Przygotowanie pendrive instalacyjnego**

### Windows – Rufus:

1. Pobierz Rufus.

```bash
diskutil unmountDisk /dev/diskN  # macOS
```

3. Nagraj ISO:

```bash
sudo dd if=/ścieżka/debian-12.iso of=/dev/sdX bs=4M status=progress oflag=sync  # Linux
sudo dd if=/ścieżka/debian-12.iso of=/dev/rdiskN bs=4m status=progress  # macOS
sync
```

# **3️⃣ Instalacja Debiana 12**

1. Włóż pendrive i uruchom komputer.
2. W BIOS/UEFI wybierz boot z USB.
3. Wybierz **Graphical install** (GUI).
4. Kroki instalatora:

   * Język → polski (lub angielski)
   * Region, strefa czasowa
   * Klawiatura
   * Partycjonowanie dysku:

     * *Guided – use entire disk* dla początkujących
     * Lub ręczne, jeśli chcesz własne partycje
   * Ustawienie użytkownika i hasła
   * Instalacja pakietów:

     * Zaznacz **Debian desktop environment**
     * Wybierz środowisko: **XFCE** (lekki, dobre na serwer + GUI)
     * Standard system utilities (SSH server możesz zaznaczyć)

# Krótkie przypomnienie przed startem

Adres do połączenia z serwerem to zwykle:

* **W LAN**: `IP_wewnętrzne_serwera:25565` (np. `192.168.1.50:25565`)
* **Przez Internet**: `IP_zewnętrzne:25565` (albo domena + port)

Z sprawdzeniem zewnętrznego IP:

```bash
curl ifconfig.me
```

---

## Opcja A — **Port forwarding** (najczęściej używane, gdy masz dostęp do routera)

To klasyczna metoda: przekierowujesz port 25565 na routerze na adres Twojego serwera.

1. **Ustal stały adres IP serwera**

   * Najprościej: ustaw w routerze *DHCP reservation* (przypisz stały adres do MAC serwera).
   * Alternatywnie statyczne ustawienie w systemie, ale łatwiej przez router.

2. **W routerze ustaw przekierowanie portu (Port Forward / Virtual Server)**

   * Port zewnętrzny: **25565** (TCP) → adres wewnętrzny: **192.168.x.x:25565** (TCP).
   * Zapisz ustawienia i zrestartuj router jeśli potrzeba.

3. **Upewnij się, że firewall na Debianie przepuszcza port**
   Jeśli używasz `ufw`:

   ```bash
   sudo ufw allow 25565/tcp
   sudo ufw reload
   ```

   Jeśli nie masz ufw, sprawdź `iptables`/`nftables`.

4. **Test**

   * Sprawdź z zewnątrz (np. kolega łączy się z Twoim publicznym IP:25565).
   * Możesz też sprawdzić dostępność portu z narzędzia online (np. „canyouseeme”) — poproś kogoś spoza Twojej sieci, żeby testował.

**Plusy:** stabilne, małe opóźnienia.
**Minusy:** wymaga dostępu do routera i publicznego IP (czasem ISP blokuje porty).

---

## Opcja B — **ZeroTier (VPN P2P)** — działa bez przekierowywania portów

ZeroTier tworzy wirtualną sieć — serwer i gracze dołączają do tej samej sieci i łączą się po adresie wirtualnym.

1. **Zainstaluj ZeroTier na Debianie** (jednolinijkowa instalacja):

```bash
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join TWÓJ_NETWORK_ID
```

2. Zaloguj się na [https://my.zerotier.com](https://my.zerotier.com) i zaakceptuj urządzenie serwera (jeśli masz konto).
3. Gracze instalują ZeroTier i dołączają do tego samego `NETWORK_ID`.
4. Po dołączeniu każdy dostaje wirtualny adres (np. `10.147.17.x`). Gracze łączą się w Minecraft po tym adresie: `10.xxx.xxx.xxx:25565`.

**Plusy:** proste, działa przez NAT, bez routera.
**Minusy:** wymaga, żeby każdy z graczy zainstalował ZeroTier; dodatkowa warstwa sieciowa.

---

## Opcja C — **Reverse SSH tunnel** (gdy masz VPS z publicznym IP)

Jeśli nie możesz przekierować portu i nie chcesz instalować VPN, możesz użyć VPS jako „przekaźnika”.

1. Zakładamy, że masz VPS z publicznym IP i konto `user@vps_ip`.
2. Na serwerze Minecraft uruchom tunel:

```bash
ssh -N -R 25565:localhost:25565 user@VPS_PUBLIC_IP
```

Wyjaśnienie: ruch na porcie 25565 na VPS będzie przekierowany do Twojego serwera lokalnego.
3. Na VPS musisz mieć w `/etc/ssh/sshd_config`:

```
GatewayPorts yes
```

i zrestartować sshd (`sudo systemctl restart sshd`) — wtedy inne komputery będą mogły łączyć się na `VPS_PUBLIC_IP:25565`.

**Plusy:** działa jeśli masz VPS.
**Minusy:** wymaga VPS i konfiguracji ssh; może być wolniejsze niż bezpośrednie połączenie.

---

## Opcja D — **ngrok (szybkie i proste, dla testów / krótkich sesji)**

ngrok pozwala wystawić lokalny serwer przez tunel. Do Minecrafta potrzebujesz trybu TCP (część funkcji może być płatna).

1. Pobierz ngrok, zarejestruj konto i ustaw `authtoken`.
2. Uruchom:

```bash
./ngrok tcp 25565
```

ngrok poda adres typu `0.tcp.ngrok.io:xxxxx` — gracze łączą się pod tym adresem.

**Plusy:** bardzo proste testowo.
**Minusy:** w darmowej wersji adresy mogą się zmieniać; nie dla produkcyjnych serwerów bez płatnego planu.

---

## Bezpieczeństwo i dobre praktyki

* **Zrób kopię zapasową** świata i plików `server.properties`, `whitelist.json`, itp.
* Używaj **whitelisty** żeby tylko zaproszeni gracze mogli wejść:

  * W `server.properties` ustaw `white-list=true` i dodaj graczy komendą na serwerze:

    ```
    whitelist add NickGracza
    ```
* Monitoruj logi (`logs/latest.log`) i używaj opcji `ops.json` do przyznawania uprawnień.
* Aktualizuj JDK, system i plik serwera.

---

## Która opcja najlepsza dla Ciebie?

* Jeśli masz dostęp do routera i publiczne IP → **Port forwarding** (Opcja A).
* Jeśli nie chcesz dotykać routera, a gracze nie mają publicznych IP → **ZeroTier** (Opcja B).
* Jeśli masz VPS → **Reverse SSH** (Opcja C).
* Na szybką próbę z nieznajomymi → **ngrok** (Opcja D).

## 🧱 Instalacja serwera Minecraft na Debianie

### 🔧 Wymagania wstępne

* System: **Debian 11 / 12**
* Uprawnienia administratora (**sudo** lub root)
* Zainstalowana Java (JDK 17 lub nowsza – wymagana przez najnowsze wersje Minecrafta)

---

### 1️⃣ Aktualizacja systemu

Otwórz terminal i wpisz:

```bash
sudo apt update && sudo apt upgrade -y
```

---

### 2️⃣ Instalacja Javy

Zainstaluj OpenJDK 17:

```bash
sudo apt install openjdk-17-jre-headless -y
```

Sprawdź wersję:

```bash
java -version
```

Powinno pokazać coś jak:

```
openjdk version "17.x.x"
```

---

### 3️⃣ Utwórz katalog serwera

```bash
mkdir ~/minecraft
cd ~/minecraft
```

---

### 4️⃣ Pobierz plik serwera (np. oryginalny Mojang)

Otwórz [https://www.minecraft.net/en-us/download/server](https://www.minecraft.net/en-us/download/server)
Skopiuj link do pliku `.jar`, np.:

```
server.jar
```

Pobierz go:

```bash
wget https://pobierz.link/do/server.jar -O server.jar
```

*(wklej prawdziwy link z oficjalnej strony)*

---

### 5️⃣ Uruchom po raz pierwszy, żeby wygenerować pliki

```bash
java -Xmx2G -Xms1G -jar server.jar nogui
```

Zobaczysz komunikat o **EULA**.

---

### 6️⃣ Zaakceptuj licencję

Edytuj plik `eula.txt`:

```bash
nano eula.txt
```

Zmień:

```
eula=false
```

na

```
eula=true
```

Zapisz i wyjdź: **Ctrl+O**, potem **Enter**, następnie **Ctrl+X**.

---

### 7️⃣ Uruchom serwer ponownie

```bash
java -Xmx2G -Xms1G -jar server.jar nogui
```

---

### 8️⃣ (Opcjonalnie) Utwórz skrypt startowy

Aby łatwo uruchamiać serwer:

```bash
nano start.sh
```

Wklej:

```bash
#!/bin/bash
java -Xmx2G -Xms1G -jar server.jar nogui
```

Zapisz i nadaj uprawnienia:

```bash
chmod +x start.sh
```

Uruchamiasz potem komendą:

```bash
./start.sh
```

---

### 9️⃣ (Opcjonalnie) Uruchamianie w tle (screen)

Aby serwer działał po zamknięciu terminala:

```bash
sudo apt install screen -y
screen -S minecraft
./start.sh
```

Aby wyjść z sesji bez wyłączania:

```
Ctrl + A, potem D
```

Aby wrócić:

```bash
screen -r minecraft
```

---

### 🔟 Otwórz port 25565

Jeśli używasz firewalla (np. ufw):

```bash
sudo apt install ufw -y
sudo ufw allow 25565/tcp
sudo ufw enable
```

<div style="float: right"><a href="#top"><span style="font-size: x-large">&uarr;</span> góra</a></div><h2>Przekierowania<a name="r__data_przekierowania_txt"></a></h2>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>></b>
          &ndash;
          <span>przekierowanie wyjścia z programu do pliku.</span>
      </dt>
      <dd>
          <p>Standardowym wyjściem każdego programu jest ekran (konsola tekstowa) a standardowym wejściem jest klawiatura. Można te wejścia i wyjścia przekierowywać dowolnie.</p>
        <ul>
            <li><i>echo "ala ma kota"  >  plik.txt</i> &ndash; wyjście z programu echo wpisze do pliku plik.txt</li>
            <li><i>ls -l  >  lista.dat</i> &ndash;  wypisze listę plików do pliku lista.dat</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>>></b>
          &ndash;
          <span>doklejenie wyjścia z programu na koniec pliku</span>
      </dt>
      <dd>
        <ul>
            <li><i>echo "ala ma psa"  >>  plik.txt</i> &ndash; dopisze "ala ma psa" na koniec pliku plik.txt</li>
            <li><i>ls -l  >  lista.dat</i> &ndash;  wypisze listę plików do pliku lista.dat</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>|</b>
          &ndash;
          <span>przekierowanie wyjścia jednego programu na wejście drugiego</span>
      </dt>
      <dd>
        <ul>
            <li><i>cat plik.txt | wc -l</i> &ndash;  cat wypisuje plik.txt na wyjście które przekierowujemy na program liczący wiersze.</li>
            <li><i>ls -l | lpr</i> &ndash; program drukujący 'lpr' dostanie na wejście listę plików</li>
            <li><i>cat plik.txt | tac | grep "coś" | head > cosie.txt</i> &ndash;  wypisanie pliku.txt na program 'tac', który odwraca kolejność wierszy, wynik tego przekierowany na 'grep', który wypisze tylko linie zawierające słowo "coś", wynik tego wysłany na program 'head', który pośle dalej tylko pierwsze 10 wierszy na wyjście, które przekierowaliśmy do pliku cosie.txt.</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>>!</b>
          &ndash;
          <span>przekierowanie do pliku. Działa podobnie jak >, ale kontynuuje nawet gdy plik już istnienie. Działa w "tcsh".</span>
      </dt>
      <dd>
        <ul>
            <li><i>echo "ala ma kota" > plik.txt</i> &ndash;  gdy plik.txt istnieje, ta komenda może się nie powieść.</li>
            <li><i>echo "ala ma kota" >! plik.txt</i> &ndash;  konieczne będzie użycie wykrzyknika >!</li>
            <li><i>echo "ala ma kota" >| plik.txt</i> &ndash;  to samo tylko w 'bash'</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b><</b>
          &ndash;
          <span>przekierowanie pliku jako standardowego wejścia.</span>
      </dt>
      <dd>
        <ul>
            <li><i>szachy < ruchy.txt</i> &ndash;  jeśli program szachy przyjmuje ruchy na standardowe wejście, możemy te ruchy spisać do pliku i podać programowi w ten sposób</li>
            <li><i>cat ruchy.txt | szachy</i> &ndash;  to samo można też zrobić tak</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b><<</b>
          &ndash;
          <span>podanie na wejście następujących później linii.</span>
      </dt>
      <dd>
        <ul>
            <li><i>szachy << DO_KONCA<br/>E2-B4<br/>H5-A1<br/>C6-F5<br/>DO_KONCA</i> &ndash;  podanie tzw. dokumentu w miejscu. Wszystkie następujące linie między etykietami 'DO_KONCA' będą podane na standardowe wejście programu 'szachy'.</li>
            <li><i>echo "E2-B4<br/>H5-A1<br/>C6-F5" | szachy</i> &ndash;  to samo można też zrobić tak</li>
            <li><i>echo -e "E2-B4\nH5-A1\nC6-F5" | szachy</i> &ndash;  to samo można też zrobić tak</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>2></b>
          &ndash;
          <span>przekierowanie standardowego wyjścia dla błędów do pliku. Oprócz standardowego wyjścia i wejścia, każdy program ma jeszcze standardowe wyjście dla błędów. Je też możemy przekierowywać, na przykład w inne miejsce niż wyjście zwykłe. Działa w "bash", nie w "tcsh".</span>
      </dt>
      <dd>
        <ul>
            <li><i>find -name "plik.*" >znalezione.log 2>bledy.log</i> &ndash;  pliki znalezione przez 'find' wylądują w znalezione.log, komunikaty o błędach nie zaciemnią nam wyniku i wpiszą się do innego pliku &ndash; bledy.log</li>
            <li><i>cp -r dane/ backup/ 2>error.log</i> &ndash;  jeśli podczas kopiowania całego katalogu wystąpią błędy, wszystkie komunikaty zostaną wpisane do error.log</li>
            <li><i>( ls > plik.log ) >& plik.err</i> &ndash;  w 'tcsh' nie można przekierować samego wyjścia dla błędów, stąd konieczność takiej konstrukcji.</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>&> lub >&</b>
          &ndash;
          <span>przekierowanie obu wyjść do pliku.</span>
      </dt>
      <dd>
        <ul>
            <li><i>ls >& plik.log</i> &ndash; standardowe wyjście i standardowe wyjście dla błędów jest przekierowane do plik.log</li>
            <li><i>ls >plik.log 2>&1</i> &ndash; to samo, ale działa tylko w 'bash'. Przekierowanie wyjścia, a potem skopiowanie go na wyjście dla błędów.</li>
            <li><i>ls &> plik.log</i> &ndash;  to samo co >& ale w notacji bardziej właściwej dla 'bash'.</li>
        </ul>
      </dd>
    </dl>

<div style="float: right"><a href="#top"><span style="font-size: x-large">&uarr;</span> góra</a></div><h2>Urządzenia systemowe<a name="r__data_urzadzenia_txt"></a></h2>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/null</b>
          &ndash;
          <span>studnia bez dna. Urządzenie do którego możemy pisać do woli, a wszystko co wpiszemy przepada.</span>
      </dt>
      <dd>
        <ul>
            <li><i>find -name "plik.*" 2>/dev/null</i> &ndash;  jeśli wsród wyników szukania nie chcemy oglądać komunikatów o błędach</li>
            <li><i>latex plik.tex >/dev/null</i> &ndash;  program wykona wszystkie czynności, ale nie zaśmieci nam konsoli logami</li>
            <li><i>licz >/dev/null 2>/dev/null &</i> &ndash;  jeśli chcemy puścić program w tle, a potem się wylogować (zamknąć konsole), musimy przekierować wyjścia z programu tak, aby nie próbował pisać do nieistniejącego już urządzenia. Urządzenie 'null' jest zawsze.</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/zero</b>
          &ndash;
          <span>składnica zer. Jest to urządzenie do czytania, które nigdy się nie kończy. Można z niego wczytać dowolną liczbę bajtów, a wszystkie będą równe zero.</span>
      </dt>
      <dd>
        <ul>
            <li><i>dd if=/dev/zero of=zera.txt count=1000</i> &ndash;  wczyta tysiąc zer do pliku 'zera.txt'.</li>
            <li><i>cat /dev/zero</i> &ndash;  radzę nie próbować</li>
            <li><i>head -c 10 /dev/zero > zera.txt</i> &ndash;  wypisze pierwsze 10 bajtów z '/dev/zero' do pliku 'zera.txt'. W wyniku mamy plik z dziesięcioma zerami.</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/random</b>
          &ndash;
          <span>zbiór liczb losowych. Jest to urządzenie do czytania, które daje prawdziwie losowe dane. Korzysta przy tym z systemowego zbiornika entropii, który jest uzupełniany dzięki różnym przejawom aktywności użytkownika. Zbiór ten może się wyczerpać, dlatego nie należy z niego czytać wielu liczb na raz.</span>
      </dt>
      <dd>
        <ul>
            <li><i>od -t x1 -N 100 /dev/random</i> &ndash;  wypisze pierwsze 100 losowych bajtów z urządzenia /dev/random na ekran (w systemie szesnastkowym)</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/urandom</b>
          &ndash;
          <span>zbiór liczba pseudolosowych. Jest to urządzenie do czytania, które podaje liczby pseudolosowe. Ma ich do dyspozycji dowolnie dużo.</span>
      </dt>
      <dd>
        <ul>
            <li><i>od -t d1 -N 100 /dev/urandom</i> &ndash;  wypisze pierwsze 100 bajtów z urządzenia /dev/urandom na ekran (w systemie dziesiętnym)</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/stdin</b>
          &ndash;
          <span>standardowe wejście aktualnego procesu. Każdy proces który próbuje czytać z tego urządzenia, dostanie zawartość własnego wejścia.</span>
      </dt>
      <dd>
        <ul>
            <li><i>echo "ma kota" | cat ala.txt /dev/stdin</i> &ndash;  program cat połączy zawartość pliku ala.txt z tym co dostał na standardowe wejście</li>
        </ul>
      </dd>
    </dl>

<hr class="komenda"/><!-- +++++++++++++++++++++++++ -->

    <dl class="kom">
      <dt>
          <b>/dev/stdout</b>
          &ndash;
          <span>standardowe wyjście aktualnego programu. Gdy proces wypisze coś do pliku /dev/stdout, pojawi się to na jego standardowym wyjściu.</span>
      </dt>
      <dd>
        <ul>
            <li><i>a2ps --output plik.ps     plik.txt</i> &ndash;  program a2ps stworzy dokument postscript w pliku plik.ps</li>

      </dd>
    </dl>

    <dl class="kom">

          &ndash;

      </dt>
      <dd>
      </dd>

## **1. Co to są komendy w Minecraft?**

Komendy w Minecraft to **specjalne polecenia wpisywane w grze**, które pozwalają Ci zmieniać świat, zarządzać graczami, mobami, blokami i innymi elementami gry w sposób natychmiastowy.
Można je wpisywać w:

* czacie (wpisujesz `/komenda`)
* command blockach (blokach specjalnie do automatyzacji komend)

Komendy pozwalają na **praktycznie wszystko**, od teleportacji, przez tworzenie struktur, po generowanie efektów specjalnych i niestandardowych mobów.

---

## **2. Po co używa się komend?**

Komendy są użyteczne w wielu sytuacjach:

1. **Dla gracza** – szybko zdobywasz przedmioty, zmieniasz pogodę, teleportujesz się lub ustawiasz punkt odrodzenia.
2. **Dla map twórców i serwerów** – tworzenie przygodowych map, minigier, wyzwań i automatycznych systemów.
3. **Dla command blocków** – można tworzyć automatyczne zdarzenia w grze, np.:

   * teleportacja mobów w określone miejsce
   * spawn lootu w chestach
   * pokazywanie napisów i efektów wizualnych
   * systemy punktów i nagród w minigrach
4. **Dla eksperymentów** – testowanie funkcji, modyfikowanie świata bez granic, tworzenie machin redstone z automatyką

---

## **3. Przykłady, co możesz robić dzięki komendom**

### **Teleportacja i ruch**

* Teleportacja gracza lub mobów w dowolne miejsce:

```
/tp @p 100 64 -200
```

* Tworzenie „portali” lub automatycznych przesunięć graczy w mapach przygodowych.

---

### **Generowanie przedmiotów i mobów**

* Daj graczowi dowolny przedmiot z enchantami i właściwościami:

```
/give @p minecraft:diamond_sword{Enchantments:[{id:"minecraft:sharpness",lvl:5}]}
```

* Przywołanie custom mobów:

```
/summon minecraft:zombie ~ ~ ~ {CustomName:"\"Strażnik\""}
```

* Możesz tworzyć bossy, niestandardowe moby, a nawet niewidzialne NPC do map przygodowych.

* **Opis:** Daje graczowi przedmiot w określonej ilości i z opcjonalnymi właściwościami NBT.
* **Składnia:**

```
/give <gracz> <przedmiot> [ilość] [data] [NBT]
```

* **Argumenty:**

  * `<gracz>` – nazwa gracza lub selector (`@p`, `@a`, `@r`, `@e`)
  * `<przedmiot>` – np. `minecraft:diamond_sword`
  * `[ilość]` – liczba przedmiotów (domyślnie 1)

  * `[NBT]` – właściwości przedmiotu, np. enchanty, nazwa, lore

### 2. `/tp`

* **Opis:** Teleportuje gracza lub mob do innego gracza lub współrzędnych.
* **Składnia:**

```
/tp <cel>

/tp <gracz> <x> <y> <z> [yaw] [pitch]
```

* **Argumenty:**

  * `<cel>` – gracz lub mob, do którego ma nastąpić teleportacja
  * `<x y z>` – współrzędne docelowe
  * `[yaw pitch]` – kierunek, w którym będzie patrzył gracz
* **Przykład:**

```
/tp @p 100 64 -200 90 0
```

Teleportuje gracza do współrzędnych X=100, Y=64, Z=-200 i ustawia go w kierunku 90° yaw i 0° pitch.

---

### 3. `/summon`

* **Opis:** Przywołuje moby lub custom entity w wybranym miejscu z opcjonalnymi właściwościami NBT.
* **Składnia:**

```
/summon <mob> [x y z] [dataTag]
```

* **Argumenty:**

  * `<mob>` – np. `minecraft:zombie`, `minecraft:armor_stand`
  * `[x y z]` – współrzędne spawnienia
  * `[dataTag]` – właściwości mobów, np. wygląd, enchanty, AI
* **Przykład:**

```
/summon minecraft:armor_stand ~ ~ ~ {CustomName:"\"MójStojak\"",NoGravity:1b,Invisible:1b}
```

Tworzy niewidzialny armor stand z nazwą „MójStojak”, który nie spada.

---

### 4. `/execute`

* **Opis:** Najpotężniejsza komenda do wykonywania złożonych akcji i warunków.
* **Składnia:**

```
/execute [as <target>] [at <target>] [positioned <x y z>] [rotated <yaw pitch>] [if|unless <warunek>] run <komenda>
```

* **Przykład:**

```
/execute as @e[type=zombie] at @s if block ~ ~-1 ~ minecraft:grass_block run tp @s ~ ~10 ~
```

Każdy zombie stojący na trawie zostanie teleportowany 10 bloków w górę.

Mannequins
25w36b
Added a new type of technical entity called minecraft:mannequin which can only be spawned with summon commands.

A Mannequin is a Player Avatar without a connected Player
Mannequins always show an "NPC" text where a Player's below_score score would show
Mannequins function as Living Entities - they can hold and wear equipment, have attributes & effects, take damage, etc
Data Fields:

profile - Which profile to show on the entity. This is either a Player profile (same format as a minecraft:profile component) or a Mannequin profile with the following fields:
texture - Namespaced ID of the skin texture to use
The skin is specified relative to the textures folder and with a .png suffix
e.g. entity/player/wide/steve will use the default wide Steve skin
cape - Optional namespaced ID of the cape texture to use
When specified, uses the same format as the texture field
Omitting this field means the Mannequin has no cape
elytra - Optional namespaced ID of the elytra texture to use
When specified, uses the same format as the texture field
Omitting this field means the Mannequin uses the cape texture, or if that is also omitted, the default Elytra texture when wearing Elytra
model - Optional model type, one of wide and slim
If omitted, wide is used
hidden_layers - List of outer skin layers to hide
Valid entries: cape, jacket, left_sleeve, right_sleeve, left_pants_leg, right_pants_leg, hat
main_hand - Which hand is the main hand of the Mannequn - one of left and right
